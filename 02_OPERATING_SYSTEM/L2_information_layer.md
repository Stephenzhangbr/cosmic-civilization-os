# Layer 2 — Information Layer (L2)

## Overview

The Information Layer defines how reality is represented, verified, and protected from manipulation.

This layer is the foundation of all higher-level systems (NCM, Governance, Coordination).

If this layer fails, the entire civilization system collapses into simulated reality.

---

## Core Principle

> Not all data is reality.  
> Only verifiable, physically anchored data can enter the system.

---

## 1. Definition of Real Data

Real Data must satisfy at least one of the following conditions:

### 1. Physical Anchor

Data must correspond to a measurable physical event:

- energy consumption  
- material transformation  
- biological signals  
- hardware state changes  

---

### 2. Irreversibility

The data must be linked to a process that cannot be perfectly reversed:

- entropy increase  
- resource depletion  
- time-bound physical processes  

---

### 3. Multi-Source Consistency

The data must be independently observable from multiple sources:

- different sensors  
- different locations  
- different system architectures  

---

## Invalid Data (Rejected by Default)

The following are NOT considered real:

- purely simulated outputs  
- unverifiable AI-generated content  
- data without physical linkage  
- single-source unverified inputs  

---

## Key Insight

> Data becomes "real" only when it resists manipulation.
> ### 4. Cross-Domain Correlation (Optional Enhancement)

Real data gains higher reliability when it produces consistent effects across multiple physical domains.

Example:

- energy systems  
- thermal signals  
- spatial movement  

The more independent physical domains a data point affects, the harder it becomes to falsify.
## Input Control Principle

No automated system is allowed to inject data into Layer 2 without verification.

All inputs must pass:

- physical anchoring  
- or multi-source validation  

Unverified automated outputs are treated as non-real by default.
